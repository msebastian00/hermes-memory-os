## Where Things Stand

The semantic retrieval milestone is complete and committed. Hermes Memory OS now supports:

- Config-required local storage through `HERMES_MEMORY_HOME`, `--data-dir`, or YAML config.
- SQLite-backed durable memories, raw events, source metadata, retrieval logs, injection logs, feedback, extraction candidates, and learning events.
- Long-form source ingestion for Markdown, plain text, EPUB, SRT, VTT, JSON transcripts, and dependency-gated PDF.
- Source chunk metadata for source type, heading, chapter, section, page, timestamp, speaker, chunking version, embedding provider/model, indexing state, and Qdrant point ID.
- Optional semantic indexing with Ollama embeddings and Qdrant.
- Hybrid retrieval that merges Qdrant semantic hits with SQLite FTS hits and falls back to FTS when semantic services fail.
- Candidate review CLI commands.
- Chief of Staff deterministic Markdown draft workflows.

## Key Commands

Basic local setup:

```bash
export PYTHONPATH=src
export HERMES_MEMORY_HOME=/tmp/hermes-memory-os-dev
python -m hermes_memory_os.cli init
python -m hermes_memory_os.cli doctor
```

Ingest and search:

```bash
python -m hermes_memory_os.cli ingest --path /path/to/wiki-or-sources
python -m hermes_memory_os.cli ingest --path /path/to/book.txt --source-type book
python -m hermes_memory_os.cli ingest --path /path/to/transcript.srt
python -m hermes_memory_os.cli search "query text"
python -m hermes_memory_os.cli search "query text" --source-type book
```

Memory add:

```bash
python -m hermes_memory_os.cli add \
  --type fact \
  --scope system \
  --summary "Summary" \
  --text "Canonical durable memory text."
```

Candidate review:

```bash
python -m hermes_memory_os.cli candidates create \
  --source-event-id evt_123 \
  --type fact \
  --scope system \
  --summary "Candidate summary." \
  --text "Candidate canonical text."

python -m hermes_memory_os.cli candidates list --status pending_review
python -m hermes_memory_os.cli candidates update cand_123 --status approved
```

Chief of Staff drafts:

```bash
python -m hermes_memory_os.cli chief process-inbox --path /path/to/inbox
python -m hermes_memory_os.cli chief daily-brief
python -m hermes_memory_os.cli chief weekly-connections
python -m hermes_memory_os.cli chief self-review
```

By default, Chief drafts write to:

```text
${HERMES_MEMORY_HOME}/vault/chief
```

Use `--output-dir` to override.

## Important Design Boundaries

- SQLite remains the source of truth.
- Qdrant stores semantic points and payloads, but SQLite hydrates retrieval results.
- Source documents are external knowledge, not personal durable memories.
- Extraction candidates are review artifacts and are not promoted automatically.
- Chief of Staff commands generate deterministic Markdown drafts only. They do not call an LLM, modify prompts/skills, or write durable memory automatically.
- Reachy/runtime state is intentionally filtered from durable metadata.
